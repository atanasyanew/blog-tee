Installing and running

Assuming git is already installed.

Hugo is easy to install, just follow the guide from the official documentation

After all of the installation processes the site can be run with

hugo serve

This command serve the application which is now accessible at localhost:1313

Copyrighting

Typography example is shown in the another post. A new post can be create by either create new file in the ./content/post folder or by terminal.

hugo new post/YYYY-MM-DD-name.md

This command will create a new post with the default template.

Deploy

For hosting the entire project is used the free GitHub feature GitHub Pages. GitHub Pages are hosted directly from your GitHub repository, so it is possible to just edit, push, and changes are live.

In order to deploy a shell scripts is used, basically the script build the application (generates the static content) and then commit and push the changes to the hosted git-pages branch. More information about the deploy script can be found at Hugo documentation

Deployment with the script can be done with the following command execute inside git bash

 ./deploy.sh

Deployment process can be optimized more with the Continuous deployment technique. There are serval Continuous deployment providers that the project can be integrate with, such as AzureDevOps,and Forestry.io

Typical use case

go to source folder run the project… hugo serve create new post save commit push

execute deploy script
